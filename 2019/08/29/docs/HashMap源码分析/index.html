HashMap底层数据结构

在JDK1.8之前，HashMap使用数组和链表结合，就是当发生冲突时，直接插入同一个链表后面，JDK 1.8之后，采用链表和红黑树作为冲突的解决方案，在链表树大于8之后转换成红黑树，在红黑树节点小于6之后转换成链表。

  • loadFactor加载因子是控制数据疏密程度。loadFactor越接近1，代表数据越密，太大导致数据查找效率低，太小数组利用率比较低。JDK官方给的是0.75f，是一个比较合适的临界值。

  • threshold，threshold=loadFactor capacity。 默认是16 0.75=12。也就是说当数据量达到12时，我们就需要考虑对数组进行扩容了。

构造方法

HashMap有四个构造函数

put方法

HashMap中put方法用于存入元素，在put方法中
调用putVal插入元素，putVal只是给put方法中调用，不提供给用户调用。

puVal主要过程。

  • 根据key定位到数组中的位置，如果没有元素直接插入
  • 如果有元素，比较key，如果相同就直接覆盖，如果不相同判断后续是否是树节点，如果是调用树方法，如果是链表直接插入链表。
